Steady-state code language

Introduction

When you work with non-linear models, the very first problem you face is finding their steady states. In IRIS, you have two options. You can use the sstate function, which attempts to find the steady-state of a model object as it is using the Optimization Toolbox's numerical methods. This approach may not be, however, always robust enough, especially in the case of larger models. The other option is to write a separate steady-state code file, manually breaking down the model's steady state into smaller blocks, and then combine symbolic and numerical methods to solve a sequence of simpler systems of equations.

Remember that although the syntax for the steady-state code files is quite similar to that used in the model code files, the logic and the meaning of the kewywords is rather different.

Steady-state code language

!parameters

List of input parameters.

Description

This keyword must appear as the first thing in each steady-state code file, and it may not appear anywhere else. The keyword is supposed to be followed by list of all parameters and other values (such as numeraires or helper variables) that are needed to compute the steady state, separated by commas, semicolons, or new lines. In other words, the resulting steady state will be a function of these input parameters only.

Example

!parameters
   alpha, beta, gamma, P

The steady-state values solved for in subsequent !equations blocks will be, in the end, functions of these four values only.
