IRIS command line help

Use IDOC to get help on IRIS functions.

Description

You can use the idoc command to invoke a help window for an IRIS function. Add the name of the class or topic (model, VAR, tseries, dates, data, container) followed by a slash or a dot, and the function name.

The idoc command is available for a limited range of functions at the moment. For other functions, you can use the older help command.

Example

To get help for the model.acf function, type

idoc model.acf
.

Optional input arguments

Most IRIS functions have a number of optional input arguments you can modify or leave unchanged..

Description

IRIS functions have usually two types of input arguments: required input arguments and optional input arguments.

  • The required input arguments always come first and in fixed order. You must specify all required arguments when calling a particual function. You can find the list and order of required input arguments on a function's documentation page, see .
  • The optional input arguments follow after the required input arguments. You can specify some, all, or none of them. To pass in optional argument, you enter them as name-value pairs: first type the name of an optional argument in single quotes (i.e. as a character string), and then the desired value separated by a comma. Omitted optional arguments will be used with their default values found in the documentation; note that the default values for optional input arguments are always underlined in the list of their possible values.

Example

The model.acf function has five optional arguments. One of them is 'order': This option specifies the order up to which the autocovariance and autocorrelation funtions are to be computed. The default value is zero.

If you therefore call this function without any optional argument,

[C,R,list] = acf(m);

then C and R will be 2-dimensional matrices with contemporaneous covariances and correlations, respectively (provided the model object m has just one single parameterisation). To compute higher-order autocovariances and autocorrelations, you need to call the function as follows:

[C,R,list] = acf(m,'order',3);

in which case C and R will be 3-dimensional matrices with 4 pages along 3rd dimension: contemporaneous covariances and correlations, and then the first-order, second-order, and third-order ones.

    Writing equations

    Basic rules for writing equations in IRIS model codes.

    Description

    The basic rules for writing equations are as follows:

    • There are two basic types of equations: transition (aka endogenous), and measurement. Put very simply, the transition equations describe the dynamics of the model, i.e. the law sof motion for transition variables. The measurement equations describe the mapping between the transition variables and the observed (measured) quantities.
    • Any model must have at least one transition equation (and hence at least one transition varable). Of course, the number of transition equations must match the number of transition variables. The same for measurement equations and variables.
    • You must end each equation with a semicolon. You can, on the other hand, spread equations over multiple lines.
    • You can re-arrange the LHS and RHS of your equations in any possible way, or you can also use the implicit function format, without an equal sign. In the latter case, the expression (which still needs to be ended with a semicolon) is automatically set equal to zero.
    • You can combine round (...) and square [...] brackets when writing more complex mathematical expressions. This is only for you convenience, and has no effect on anything whatsoever.
    • Time subscripts are enclosed in curly brackets, such as x{-1} for a t-1 lag, or x{2} or x{+2} for a t+2 lead. Note that each equation is implicitly conditioned by time t information: Therefore, any lead is, in fact, a conditional expectation of a future value.

    See the model code language section for details on the model code language keywords.
