  1. To start an IRIS session,
    • either run irisstartup; in the Matlab command window,
    • or press the Matlab Start button (bottom left in the Matlab window), and go to Link and Targets → The IRIS Toolbox → Start IRIS session.

    If Matlab failed to save IRIS on the permanent search path during installation, you will need to run the following two commands each time you want to start working with IRIS: addpath C:\IRIS_Tbx; irisstartup; where you replace C:\IRIS_Tbx with your IRIS directory.

  2. To finish an IRIS session, and temporarily remove IRIS files from the Matlab search path,
    • either run irisremove; in the Matlab command window,
    • or press the Matlab Start button (bottom left in the Matlab window), and go to Link and Targets → The IRIS Toolbox → Stop IRIS session.
  3. IRIS automatically detects TeX/LaTeX (using TeX's standard kpsewhich program), provided it is properly installed on your computer. There is no need to manually link TeX/LaTeX with IRIS any longer.
  4. Courtesy of the U.S. Census Bureau, the X12-ARIMA program, Version 0.3 Build 177, is incorporated and redistributed with IRIS. There is no need to manually download, install and link X12 with IRIS any longer.
  5. You can customise the Matlab Editor so that your IRIS files (such as model codes, steady-state codes, and quick-report files) get syntax-highlighted. Close the Matlab Editor, open Matlab's File → Preferences → Editor/Debugger menu and go to the Language options. Make sure that the very first Language pop-up menu at the top of the window says "M". At the bottom of this window, you can now add the extensions you want to use with your IRIS model codes; they will appear underneath the standard "m". You can, in fact, add any extensions you will use for IRIS model codes, steady-state codes, and quick-report files (such as "model", "sstate", "q"). Note that there is no restriction on which extensions you use with your model code filenames in IRIS.
